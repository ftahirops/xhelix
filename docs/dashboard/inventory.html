App Inventory Fingerprinting

On bootstrap and on change, the endpoint sends a Class A fingerprint of the host's OS + packages + service versions + config hashes. The hub uses this to compute ProfileKeys correctly.

Fingerprint layers

OS layer

  • Distro + version + kernel + BTF + libc + systemd version
  • Package manager state hash
  • CVE-feed match delta against installed versions

Installed packages

Full list via dpkg-query -W / rpm -qa / apk info -v with versions.

Running services

Per service: binary path + SHA-256 + version (queried via --version) + modules/extensions loaded.

Config hashes

SHA-256 of: nginx.conf+Includes, sshd_config, my.cnf, php-fpm pool configs, postfix/dovecot configs, systemd unit files for managed services, crontab files.

Derived ProfileKey

Endpoint computes ProfileKey locally; reports it to hub. Hub buckets endpoints into fleet-consensus groups.

Privacy
All fingerprints are version + hash only. No request data, no log contents, no secrets. Audit via xhelixctl telemetry preview.
