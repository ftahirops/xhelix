Behavioral Reference Profiles

Signed, version-pinned, config-aware profiles describing what each app should legitimately do. Cached locally on endpoint; updated only via curator-signed releases.

Core concepts

  • ProfileKey — (app, version_family, os_family, package_origin, role, feature_fingerprint, phase). Bucketing key for fleet consensus + runtime lookup.
  • ConfigDerivedBehavior — what the operator config DECLARES: listen ports, read/write roots, allowed exec, declared upstreams, loaded modules.
  • Confidence class — Strict (exact match), StableFallback (version-family match), ConstrainedAdaptation (role-only match), Unprofiled (no match → conservative log-only).

Shipped config parsers (T03)

  • pkg/brp/parser/nginx.go — listen/upstream/lua/njs/grpc/fastcgi/scgi/uwsgi roles
  • pkg/brp/parser/apache.go — VirtualHost/Listen/ProxyPass/LoadModule/CGI/WSGI
  • pkg/brp/parser/sshd.go — Port/PermitRootLogin/AuthorizedKeysFile/Subsystem/Match
  • pkg/brp/parser/mysql.go — bind-address/datadir/log_*/plugin_load/replication
  • pkg/brp/parser/phpfpm.go — pool listen/user/pm/security.limit_extensions/php_admin_*

Signed profile schema (T05)

JSON with detached Ed25519 signature. Includes signing_epoch, sample_count, fleet_count, version_range, full Key + Behavior + parse warnings.

Protect-our-own backstop is enforced at load
A profile claiming write/exec to a path in ProtectedSystemPaths is rejected by Profile.Validate() before signature verification. See Protect-Our-Own Backstop.
