Mission & Scope

Open-source Linux server EDR that observes via eBPF, attributes via source lineage, decides via BRP + verification engine, and contains via graduated reversible ladder — without trusting the application or any single component overriding the protect-our-own safety list.

Mission statement

xhelix v2 evolves from a per-host detector into a fleet-aware, source-attributed, BRP-validated, verification-gated, causally-composed Linux server EDR with day-one protection via signed reference profiles and zero-effort operator onboarding.

Threat surface targeted

  • Strong: credential theft, kernel-level secret access, persistence implantation, process injection, rootkit primitives, C2 / beacon traffic, source-attributed delayed persistence, forensic tamper
  • Strong (post-exploit phase): webshell uploads, Log4Shell post-exploitation, SQLi-with-file-write, SSRF→IMDS, WordPress RCE outcomes
  • Moderate: sensitive data exposure (FS-side), some DoS patterns
  • Out of scope: see What xhelix is NOT

Honest ceilings

  1. Root attacker with kernel-module-load capability defeats any userspace EDR including xhelix
  2. Memory-resident implant inside a legitimate process during a credential TTL window — ~30% residual undetected
  3. Off-host attack infrastructure (CDN/proxy chains) — only as good as the threat-intel feed
  4. Application-layer logic bugs (SQL injection at parameter parsing) — needs RASP, not host EDR
