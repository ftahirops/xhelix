Telemetry Classes

Three telemetry classes determine what goes to the hub. Default: A=on, B=on, C=off. Compliance mode: all off — endpoint still works.

Class definitions

ClassDefaultPer host per dayContents
A — always sendon<1 MBAlerts, hard-invariant violations, secret-touch summaries, lineage anchors, incident edges, app inventory fingerprint
B — summarizedon<10 MBHourly aggregates: counts of benign reads/writes/outbound by class, BRP-envelope adherence %, resource percentiles
C — opt-in burstoff0 – 500 MB / incidentFull eBPF detail for 5 min around an alert, RWX memory snapshots, deep request-linked traces

Operator control

Edit /etc/xhelix/telemetry.yaml. Compliance mode (no upload) keeps endpoint fully functional — only loses hub-side cross-host analysis.

# /etc/xhelix/telemetry.yaml
classes:
  a: true     # alerts + lineage + inventory
  b: true     # hourly aggregates
  c: false    # forensic bursts on incident
