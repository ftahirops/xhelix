Forensic Chain

Ed25519-signed hash-chain of every event. Verifiable offline by xhelix-verify without any access to the daemon, hub, or curator. Tamper-evident even if the EDR itself is compromised.

Structure

Events are batched (configurable; default 1000-event batches or 10-second windows). Each batch contains:

  • The events themselves (canonical JSON)
  • SHA-256 of the previous batch (hash chain)
  • Ed25519 signature of (events + prev_hash)
  • Signing epoch + signer key id

Verification

xhelix-verify --chain /var/lib/xhelix/chain.db --pub /etc/xhelix/chain.pub

Walks the chain, re-verifies each batch signature, re-computes hash chain. Names the exact tampered batch on failure.

Court-defensibility
The forensic chain stays on the endpoint as the authoritative record. The hub gets a stream copy for searchability, but the cryptographic evidence chain is local. If the hub is later proven compromised, endpoint chain still verifies independently.
