What xhelix is NOT

Honest framing of what xhelix doesn't cover. Most application-layer attacks (XSS, SQLi, CSRF, OAuth flaws, 2FA bypass, password-reset) are invisible to a host EDR by construction.

Explicit non-goals

  • Not a WAF — no per-request HTTP inspection at application layer
  • Not a RASP — no in-process language-runtime hooks (use Aikido Zen for that)
  • Not a SAST / DAST tool — not a code or vulnerability scanner
  • Not a Windows / macOS endpoint product — Linux-only by design
  • Not a CSPM tool — no cloud misconfiguration scanning
  • Not OSINT / external recon — no view outside the host
  • Not a SIEM — though events stream to one if you have it

Coverage against typical bug-bounty findings

Of 23 categories in the Galaxy bug-bounty checklist, xhelix has meaningful coverage on ~5 (file-upload-webshell, Log4Shell, SQLi post-exploit, SSRF, WordPress RCE post-exploit). The other ~17 are pure application-layer attacks that need a WAF or RASP. See Comparison and the canonical doc docs/BUGBOUNTY_VS_XHELIX_COVERAGE_2026-05-23.md.
