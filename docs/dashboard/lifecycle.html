Data Lifecycle & Retention

Eight-stage pipeline from kernel event to long-term archive. Each stage drops volume by 10-100×.

See Diagram 6 on the Diagrams page for the visual representation.

Stages

  1. Raw eBPF — 500-2000 ev/sec, ~5 GB/day
  2. EAC dedup — ~95% dropped, ~250 MB/day
  3. Baseline cut — delta-only, ~150 MB/day
  4. Class A/B/C split — A: ~1 MB/day to hub
  5. Hub TimescaleDB — zstd compression, 10-20× ratio

Retention tiers on hub

AgeStoragePer host per year
0–24hRaw, indexed, full fidelity~150 MB
24h–30dCompressed (10×) chunks~3 GB

TOTAL ~3.5 GB/host/year

Class C (forensic burst) retention is explicit "forever" per scope decision (FINAL_SCOPE_V2 §3.4.1), revisited when measured volume informs policy.
