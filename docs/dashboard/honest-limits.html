Honest Non-Promises

What we explicitly do NOT promise. Helps you make a realistic decision.

  1. The 28-week estimate is single-engineer focused work. Distractions or technical surprises will extend it.
  2. The 85% TP / <3% FP targets are projections based on Holmes 2019 (~85%) and commercial benchmarks (CrowdStrike 75-85%). Actual measured numbers may differ by ±5 percentage points.
  3. <0.5% FP on BRP-covered apps requires active curation. Without it, FP rate degrades over time.
  4. Real adversaries adapt. TP rate erodes over time without continuous rule + invariant tuning.
  5. Root attacker with kernel-module-load capability defeats any userspace EDR — including xhelix.
  6. Memory-resident implants during a credential TTL window remain partially undetected (~30% residual).
  7. Off-host attacker infrastructure detection is only as good as threat-intel feeds.
  8. Application-layer logic bugs (SQLi at parameter parsing) need RASP, not host EDR.
  9. xhelix v2 won\x27t match SentinelOne’s UX at launch. CLI-first operator surface.
  10. xhelix won\x27t replace CrowdStrike for organisations needing Windows + SOC support + compliance certifications.
