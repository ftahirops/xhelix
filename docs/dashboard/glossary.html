Glossary

Quick definitions for the terms used throughout the docs.

Terms

BRP (Behavioral Reference Profile)
Signed JSON profile describing what an app should legitimately do. Cached locally; updated only via curator-signed releases.
CausalSet
Bounded (16) deduplicated set of SourceAnchor IDs that contributed to a process. Becomes Ambiguous when ≥2 sources merge.
credbroker fangate
The fanotify-based syscall interceptor that denies credential file reads at the kernel level (not after).
EAC (Event Admission Controller)
The dedup + ordering + backpressure stage between sensors and the pipeline.
FIM (File Integrity Monitor)
Tracks writes/changes/deletions to a configured set of paths.
LLR (Log-Likelihood Ratio)
The scoring math used by the verification engine. Replaces naive Bayes which would over-count correlated features.
PrimarySource
Most-recent contributing SourceAnchor on a process. Used for operator display and quick-score.
ProfileKey
The bucketing key (app, version_family, os_family, package_origin, role, feature_fingerprint, phase).
Protect-our-own
The immutable list of paths xhelix can never touch with its own actions — even with a correctly-signed BRP.
SourceAnchor
The unique ID minted at every authenticated entry point (SSH, PAM, sudo, cron, systemd). Persists in source.db.
Soak gate
Delay + recheck before applying containment. Prevents acting on transient signals.
PanicSwitch
Operator command that rolls back containment steps 3-9 atomically.
