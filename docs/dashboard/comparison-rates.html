Level 3 — Fact-based detection rates

The honest framing: detection-rate comparisons are EASY to fake. Real comparisons need (1) the same test corpus, (2) the same scenarios, (3) public methodology. The only public benchmark with all three is MITRE ATT&CK Evaluations. Everything else is vendor-claim or our own measurement on a custom corpus. Both are clearly labeled below.

Where each tool actually wins

If your priority is...

  • Enterprise SOC integration: CrowdStrike / SentinelOne. Mature dashboards, threat-intel feeds, 24/7 support.
  • Windows + Mac + Linux on one stack: CrowdStrike / SentinelOne / MS Defender. xhelix is Linux-only.
  • Application-layer protection (SQLi/XSS/RCE): Aikido Zen RASP. WAFs. xhelix does not stop application-layer bugs.
  • Cheapest open-source kernel monitoring: Falco for rules-only, Tetragon for enforcement.
  • Self-hosted SIEM-style log aggregation: Wazuh / Elastic.
  • Linux-server-specific deep behavioral defense: xhelix v2.

If your concern is...

  • Vendor lock-in / cost: xhelix (Apache-2.0), Falco/Tetragon (Apache-2.0), Wazuh (GPL).
  • Compliance / air-gap requirement: xhelix (no mandatory cloud), Wazuh, Falco. CrowdStrike/SentinelOne/MS Defender require cloud connectivity.
  • Forensic admissibility: xhelix (Ed25519 chain + offline verifier). No competitor offers offline-verifiable tamper-evidence.
  • Operator transparency — "why did it block this?": xhelix (explainable: BRP field violated, profile id, signer). Cloud EDRs say "ML flagged it".
  • Custom application defense: Aikido Zen (RASP at runtime) + xhelix (host post-exploit). They’re complementary.
The honest summary:
  • xhelix v2 is the strongest open-source Linux-server EDR by a wide margin once v2 ships. v1 is 73% TP / 8% FP — mid-pack vs commercial.
  • For Linux-only fleets where operator transparency and self-hosting matter, xhelix wins on architecture; commercial EDRs win on SOC tooling and mature threat intel.
  • For Windows+Mac+Linux fleets, xhelix is not the right tool. Use a commercial EDR for the others; xhelix can run alongside on Linux.
  • For web application bugs (SQL injection, XSS, OAuth flaws), use a WAF or RASP (Aikido). xhelix catches the post-exploit phase, NOT the bug itself.
  • No tool replaces good operational practice (patching, access control, secrets management). They’re force multipliers, not silver bullets.
